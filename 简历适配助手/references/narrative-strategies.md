# Narrative Strategies

Select a primary narrative before writing bullets. Use the JD persona to choose.

## Narrative A — 0→1 Builder

**Use when:** project is a full product prototype or end-to-end build.  
**Focus:** user problem definition, core flows, architecture decisions.

**Bullet spine:**
1. Defined user problem + product model
2. Designed core flows + data model
3. Built system architecture enabling the product loop

## Narrative B — System Thinker

**Use when:** architecture, scalability, reliability, or platform work dominates.  
**Focus:** trade-offs, performance strategy, extensibility.

**Bullet spine:**
1. Identified system bottleneck or constraint
2. Designed scalable architecture/data model
3. Implemented performance or reliability strategy

## Narrative C — AI / Data PM

**Use when:** AI/RAG/data pipeline is central.  
**Focus:** knowledge flow, evaluation, latency/quality trade-offs.

**Bullet spine:**
1. Defined AI experience + user intent
2. Designed data/retrieval pipeline and evaluation
3. Tuned system for reliability, latency, or quality

## Selection Rules

- Choose the narrative that best matches the top 3 JD priorities.
- If two are close, choose a primary and weave a secondary in 1 bullet.
- Avoid mixing narratives randomly; keep a single story arc.
