# Output Schema

## Mapping Summary (Pass 1 Output)

Include:
- **Ideal PM Persona**: 3–5 JD priorities and vocabulary
- **Narrative Choice**: primary (and optional secondary)
- **Top Signals**: strong signals with evidence pointers
- **Claim Map**: claim → evidence → PM framing → JD match
- **Gaps**: missing signals and what evidence would close them

## Resume Bullets (Pass 2 Output)

Use 3–5 bullets per project. Each bullet should include:

**Template A (decision + user + system):**  
“Defined [user/problem] and [product model], then designed [system/flow] to enable [capability], informed by [evidence].”

**Template B (architecture + trade-off):**  
“Chose [architecture/data model] to balance [trade-off], enabling [capability] for [user flow], based on [evidence].”

**Template C (AI/Data):**  
“Designed [AI/data pipeline] to deliver [user outcome] with [quality/latency] constraints, evidenced by [components].”

## Missing Signals

List gaps that prevent stronger PM framing:
- No measurable outcomes → suggest adding metrics
- No user definition → ask for target persona or use case
- No prioritization evidence → ask for why features were chosen
