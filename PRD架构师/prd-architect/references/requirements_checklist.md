# Requirements Checklist

Use this to ensure PRD completeness and consistency.

## Completeness

- Problem statement is clear and user-focused.
- Target user segment is defined.
- User stories exist and map to requirements.
- Functional requirements are testable.
- Edge cases are defined with expected behavior.
- Non-functional requirements include measurable thresholds.
- Success metrics are defined with targets and data sources.

## Consistency

- Goals align with success metrics.
- Requirements trace back to user stories.
- Edge cases do not contradict core behaviors.
- NFRs do not conflict with functional requirements.

## Assumptions

- Key assumptions are listed and tagged.
- Evidence is listed for any high-impact claim.
